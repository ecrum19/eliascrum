# Elias Crum Personal Website

Personal academic website built with Vue 3, Vite, Vuetify, and TypeScript.

## Release

Current release: **v0.1.0** (2026-03-02)

## Live Scope

The site currently includes:

- About page and profile overview
- CV page powered by structured data
- Publications page with filtering, details expansion, and external profile links
- Talks/Posters page with filtering, per-item details, and PDF previews
- Light/Dark theme toggle

## Tech Stack

- Vue 3
- TypeScript
- Vue Router 4
- Vuetify 3
- Vite 6
- ESLint

## Repository Structure

- `src/components/`: page and layout components
- `src/data/`: editable content/data models used by the pages
- `scripts/`: data-generation and maintenance scripts
- `public/talks/`: generated, web-served slide/poster PDFs
- `.talk-assets/`: compressed source archives for talks/posters used by `build:slides`
- `.github/workflows/`: CI, deployment, and automation workflows

## Routes

- `/about` -> About page
- `/about/cv` -> CV page
- `/work` -> Publications
- `/talks` -> Talks and posters
- `/talks/:slug` -> Talk detail
- `/blogs` -> Blogs

## Project Scripts

- `npm run dev`: start Vite dev server
- `npm run preview`: preview production build
- `npm run build`: production build (`dist/`)
- `npm run lint`: lint Vue/TypeScript source
- `npm run build:slides`: rebuild generated talks/posters data and PDF outputs
- `npm run update:scholar`: refresh Google Scholar citation counts

## Content Editing Guide

### CV

Edit structured CV content in:

- `src/data/cvData.ts`

### Publications

Edit publication metadata, abstracts, BibTeX, tags, and publication->presentation links in:

- `src/data/publicationsData.ts`

Google Scholar citation values are stored in:

- `src/data/scholarCitations.ts` (auto-generated)

### Talks and Posters

Talk display metadata (titles, dates, descriptions, venue/topic tags):

- `src/data/talkMetadata.ts`

Generated talk/poster catalog:

- `src/data/talksData.ts` (auto-generated by `build:slides`)

### Talks/Posters Source Inputs

Long-term source storage uses compressed archives:

- `.talk-assets/slides.tar.gz`
- `.talk-assets/posters.tar.gz`

You can also temporarily add raw inputs using:

- `./slides/*.pdf`
- `./posters/*.pdf`

When `npm run build:slides` runs:

1. It reads from `./slides` and `./posters` if present; otherwise from `.talk-assets/*.tar.gz`.
2. It regenerates `public/talks/slides` and `public/talks/posters`.
3. It applies Ghostscript compression to generated output PDFs when available.
4. If raw `./slides` / `./posters` were used, it archives them into `.talk-assets/*.tar.gz` and removes the raw folders.

## Automation and CI

### Build Validation

- Workflow: `.github/workflows/build-on-push.yml`
- Runs install, optional `build:slides`, lint, and build on each push.

### GitHub Pages Deployment

- Workflow: `.github/workflows/deploy-pages.yml`
- Deploys on push to `master` and on manual dispatch.
- Computes correct Vite `base` for either project pages (`/<repo>/`) or user pages (`/`).
- Adds SPA fallback (`404.html`) and `.nojekyll` to support Vue Router history mode on Pages.

### Scholar Citation Refresh

- Workflow: `.github/workflows/update-scholar-citations.yml`
- Scheduled weekly and available via manual dispatch.
- Updates `src/data/scholarCitations.ts` and auto-commits changes.

## GitHub Pages Setup Notes

In repository settings:

1. Open **Settings -> Pages**.
2. Set source to **GitHub Actions**.
3. Ensure the `deploy-pages.yml` workflow has permission to deploy.
